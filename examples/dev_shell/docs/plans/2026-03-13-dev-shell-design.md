# 开发用 Shell 设计文档

**日期**：2026-03-13  
**范围**：跨平台（Linux / macOS / Windows）、Rust 实现、虚拟文件系统、.bin 持久化、只读导出

---

## 1. 目标与整体架构

### 1.1 目标

- **跨平台**：支持 Linux、macOS、Windows。
- **不修改宿主**：运行时不改动系统文件；所有可见文件在自带的虚拟文件系统中。
- **虚拟 FS 持久化**：虚拟 FS 保存为二进制 `.bin` 文件，可携带、可复现。
- **能力**：
  - **(A)** 在隔离环境中执行脚本/命令（首版仅内置命令）。
  - **(B)** 类 Bash 交互：`cd`、`ls`、`cat`、管道、重定向等，均作用于虚拟 FS。
  - **(C)** 可携带沙盒：一个 `.bin` 即一个沙盒，换机器/系统打开同一 `.bin` 即可继续。
- **只读导出**：虚拟 FS 可**按需**以**只读**方式映射到宿主临时目录（通过内置命令触发）。

### 1.2 整体架构

- **REPL 循环**：读入一行（或脚本文件）→ 解析为命令/管道/重定向 → 在虚拟 FS 上执行内置命令 → 输出到 stdout/stderr → 继续下一行。
- **虚拟 FS 模块**：内存中的树（目录 + 普通文件）；路径内部统一为 Unix 风格（`/`）；提供解析路径、读写文件、列举目录、创建目录/文件等 API；不直接访问宿主磁盘（除加载/保存 .bin 与按需只读导出）。
- **持久化**：虚拟 FS 序列化到 `.bin`；启动时可加载；退出或显式命令可保存。
- **只读导出**：内置命令（如 `export-readonly`）将虚拟 FS 的某子树复制到宿主临时目录并打印路径；不提供写回，仅只读。
- **进程边界**：一个 Shell 进程 = 一个 REPL + 一个虚拟 FS 实例 + 可选已加载的 .bin；首版不在此进程内启动宿主上的 `cargo`/`npm` 等。

## 2. 虚拟 FS 的数据模型与 .bin 格式

### 2.1 内存模型

- **树结构**：节点分为**目录**（名字 + 子节点列表）和**文件**（名字 + 二进制内容 `Vec<u8>`）；根为目录。
- **路径**：内部统一为 Unix 风格（`/` 分隔）；解析用户输入时把 `\` 和 Windows 盘符等归一为 `/` 再解析。
- **当前工作目录**：由根起的路径或节点引用；相对路径基于 cwd 解析。

### 2.2 需要持久化的内容

- 整棵树的拓扑与每个文件的内容。
- 可选：当前工作目录路径，便于下次打开 .bin 时恢复 cwd。

### 2.3 .bin 文件格式

- **魔数 + 版本**：前若干字节标识格式（如 `DEVS`）和版本号，便于扩展。
- **目录结构**：递归或扁平表表示树；每节点含类型（目录/文件）、名字长度、名字；目录含子节点；文件含长度 + 原始字节。
- **元数据**：当前工作目录路径及将来可扩展字段（如时间戳）。
- 序列化在内存完成；读写 .bin 时仅做「打开 → 写入/读取字节 → 关闭」。

### 2.4 边界情况

- 路径中的 `..`、`.` 在解析阶段规约；根以上再 `..` 视为停在根；空路径或仅 `/` 表示根。
- 名字约束：禁止 `/` 和 NUL；可禁止或转义其他控制字符。

## 3. 内置命令、管道与重定向、跨平台

### 3.1 首版内置命令（均在虚拟 FS 上操作）

| 类别     | 命令 | 说明 |
|----------|------|------|
| 导航状态 | `pwd` | 打印当前工作目录 |
|          | `cd <path>` | 切换 cwd |
| 目录     | `ls [path]` | 列目录，支持简单选项如 `-l` |
|          | `mkdir <path>` | 递归创建目录 |
| 文件     | `cat <path> [...]` | 输出文件内容，二进制安全 |
|          | `touch <path>` | 创建空文件或更新“修改时间” |
|          | `echo [args]` | 参数写 stdout，可支持 `-n` |
| 导出     | `export-readonly [path]` | 将 path（默认 `/` 或 cwd）子树复制到宿主临时目录并打印路径，只读 |
| 持久化   | `save [path]` | 将虚拟 FS 写入 .bin |
|          | `exit` / `quit` | 退出，可选提示保存 |

首版可仅实现以上集合；`rm`、`rmdir`、`cp`、`mv` 等可后续添加。

### 3.2 管道与重定向

- **管道**：`cmd1 | cmd2`。仅在内置命令间；前一命令 stdout 作为后一命令 stdin；同一进程内用内存缓冲区或管道模拟。
- **重定向**：`cmd > path`（stdout 写虚拟 FS）、`cmd 2> path`（stderr）、`cmd < path`（stdin 从虚拟 FS 读）。解析阶段识别，执行时用虚拟 FS 文件替换 stdin/stdout/stderr。
- 管道与重定向可组合：`cmd1 < a | cmd2 > b`；解析顺序：先重定向，再管道，再执行。

### 3.3 跨平台约定

- **路径**：用户输入归一化（`\` → `/`，盘符映射）；内部与 .bin 一律 Unix 风格。
- **行尾**：REPL 接受 `\r\n` 或 `\n`；虚拟 FS 内文件为二进制；`echo` 等输出用 `\n`；Windows 控制台可选 `\n` → `\r\n`。
- **临时目录**：`export-readonly` 使用 `std::env::temp_dir()` 或跨平台库创建子目录并复制内容；注意 Windows 文件名限制。

## 4. 错误处理与测试策略

### 4.1 错误处理

- **分层**：FS 层返回 Result；REPL/命令层转为用户可读消息并设置退出码（如 1）。
- **解析**：语法错误在解析阶段报错写 stderr，不执行。
- **执行**：FS 操作失败则终止该命令并报错；管道中任一段失败则整条管道失败。
- **.bin**：加载时文件不存在则从空根开始；格式/版本错误则明确报错；保存失败时保留内存 FS 并报错。
- **export-readonly**：复制失败则报错并返回非零退出码，不部分复制。

### 4.2 测试策略

- **虚拟 FS**：单元测试覆盖路径解析、节点增删改查、边界；序列化/反序列化往返一致；.bin 在多平台读取结果一致。
- **命令**：每条内置命令用「给定 FS + 命令」测 FS 变化与 stdout/stderr；管道与重定向单独及组合测试。
- **集成**：端到端：REPL 多行/脚本输入，断言最终 FS 与 .bin；可选验证 `export-readonly` 后临时目录内容与虚拟 FS 一致。
- **跨平台**：CI 在 Linux、macOS、Windows 上运行上述测试。

## 5. 后续扩展（非首版范围）

- 将虚拟 FS 映射到临时目录后，在只读视图下执行宿主命令（如 `cargo build`），结果写回虚拟 FS 再持久化。
- 更多内置命令：`rm`、`rmdir`、`cp`、`mv`、`grep`、环境变量等。
