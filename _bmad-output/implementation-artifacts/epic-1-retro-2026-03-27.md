## 2. 故事层洞察

### 1.1～1.2（创建校验、列表与空结果）

- **1.1：** `handle_add` 在 `dry_run` 早退前执行 `patch_from_add_args`，避免非法可选字段占用新 id；失败不写盘；集成测锁定「非法 add 不改变既有 `.todo.json`」。
- **1.2：** `todo_list_json_payload` 统一空列表 JSON（`empty`/`message`）；**`cargo xtask todo --json list`** 顺序与 **argh** 解析一致。
- **审查延期项：** 损坏的 `.todo.json` 与空集在成功路径上可能不可区分；**`--dry-run` 是否校验可选参数**与 1.1 延期记录一并留给产品与后续故事。

### 1.3～1.4（列表矩阵、完成/删除）

- **棕地为主：** 核对 `list_with_options` / `handle_list` 等已满足 AC；补集成测覆盖排序、过滤空载荷、非法 flag 退出码与**不写盘**。
- **1.4：** JSON 端到端测覆盖完成/删除、id 为 0、不存在 id、**不写盘**保留等；与 **TC-A2** 类 ID 对齐。

### 1.5（详情/更新）

- **行为修正：** `handle_update` 在 **`--dry-run` 且 id 不存在**时与 complete/delete 一致返回 **`TodoCliError::Data`（退出码 3）**。
- **`show_update` 集成模块**与 `init-ai` 文案一致性核对。

### 1.6～1.7（搜索统计、导入导出）

- **1.6：** 领域与集成测锁定 **TC-T10/T11**、空关键词、**`stats` JSON** 三字段等；未改 dispatch 主行为。
- **1.7：** **`load_todos_for_import`** 与浏览用的 **`load_todos_from_path`**（缺文件空列表）区分；缺失文件/语法错误走 **General** 与约定退出码；**`todo --json --dry-run import`** 顺序写入帮助与测试。

### 1.8（重复任务）

- **复用领域规则：** `TodoList::complete` 生成下一实例；CLI 不重复实现递推逻辑。
- **集成测：** `complete` 下一实例、`complete --no-next`；与 **Epic 1** 全量回归及 **`integration`** 一致。

### 跨故事模式

- **双 crate：** 领域与类型在 **`xtask-todo-lib`**，CLI 分装在 **`xtask`**；故事反复出现 **`cargo test -p xtask-todo-lib`** + **`cargo test -p xtask`**（含 **`integration`**）。
- **可追溯性：** 多条故事明确 **`docs/test-cases.md`** ID 与集成测名称对应。
- **已知技术债（延期）：** 损坏 JSON、dry-run 校验深度等以 **Review Defer** 形式记录，不阻塞 Epic 1 **done**。
