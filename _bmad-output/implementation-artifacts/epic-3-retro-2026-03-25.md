## 2. 故事层洞察

### Story 3.1（交互式会话与脚本执行）

- **入口：** `run_main_from_args` 区分无 `-f` → REPL，有 `-f` → 脚本；与 §5.3–§5.5 一致。
- **NFR-I1：** 参数错误、脚本缺失等走 `stderr` + 非零退出，非静默成功。
- **测试边界：** 自动化覆盖 REPL/脚本、`-e`、错误用法；**SKIP** 真实 TTY 交互与 Mode P/VM 端到端（环境依赖），在 Completion Notes 中明示。

### Story 3.2（内置文件与目录操作）

- **FR15 / NFR-S2：** `run_builtin_core` 白名单分派；`sh` 字面量不当作宿主 shell 执行；`Command::new("sh")` 等仅出现在已文档化路径。
- **测试：** `cd`/`cat` 错误路径、`sh` 非白名单等集成测补强。
- **Mode P：** 单元与路径映射逻辑跑通；真实 guest 环境端到端未强制在本机执行。

### Story 3.3（管道、重定向与缓冲上限）

- **NFR-P2：** 非末段 stdout 整段缓冲，超限 `PipelineInterStageBufferExceeded`，管线中止；末段直连不受段间 16 MiB 误用。
- **测试：** `execute_pipeline_stops_on_non_terminal_stage_overflow` 覆盖超限时后续阶段不执行。
- **常量治理：** 若调整 `PIPELINE_INTER_STAGE_MAX_BYTES`，须同步 `requirements`、`devshell-vm-gamma`、CHANGELOG（故事内已约束）。

### Story 3.4（Devshell 内 todo 子集）

- **FR17：** `todo_io` + `todo_builtin` 与 xtask 共用 `TodoDto` 语义；**不支持** `export` / `import` / `init-ai`，未知子命令明确拒绝。
- **Mode P：** `.todo.json` 绑定宿主 `cwd`，与模块注释及设计 §11 一致。
- **测试：** 持久化、`todo_file` 路径、拒绝不支持子命令等用例补强。

### Story 3.5（帮助与 TTY 补全）

- **FR18 / FR19：** `run_builtin_help`、`run_builtin_core`、`BUILTIN_COMMANDS`（含 `export-readonly` 别名）三处一致；TTY 使用 `CompletionType::List`；非 TTY 不依赖 Tab 补全。
- **测试：** help 输出清单、补全候选与别名回归测。

### 跨故事模式

- **棕地策略：** 以核对 AC、补测、边界修正为主，避免无依据重写解析器或 VFS 核心。
- **审查侧车：** 多个故事的代码审查顺带统一 **`sprint-status.yaml` 文件头 `# last_updated` 与 YAML `last_updated` 字段**，减少双源时间戳。
- **与 Epic 4：** devshell 会话、VFS、内置与 `todo` 为 VM/宿主分派（Epic 4）提供上层行为基础。
