## 2. 故事层洞察

### Story 6.1（跨平台可编译交付物）

- **门禁焦点：** pre-commit 与验收强调对 **`xtask-todo-lib`** 的 **`x86_64-pc-windows-msvc`** `cargo check`；文档已澄清 **`xtask`/`todo` 二进制**不以同一条 MSVC 矩阵作为强制门禁，避免读者误解范围。
- **测试稳定性：** 记录到并行 `cargo test -p xtask-todo-lib` 曾触发既有非确定性用例；与 pre-commit 一致采用 **`--test-threads=1`** 策略，属**流程共识**而非本 Epic 新债。
- **提醒：** 勿将 **`crates/todo/.dev_shell.bin`** 等非故事产物误提交（故事 Review 已标注）。

### Story 6.2（crates.io 与 cargo devshell）

- **`cargo publish --dry-run`：** 预检通过；**「版本已在 index 存在」**类提示为**预期**，真正 bump 版本后应再跑 dry-run 留存日志（低优先级建议）。
- **安装演练：** 默认与 `--no-default-features` 两套 `cargo install` + `cargo devshell` 冒烟通过。
- **文档修正：** `docs/publishing.md` 与 **`default = ["beta-vm"]`** 对齐；**NFR-R2** 选定 **GitHub Release 说明为单一事实来源**（若日后增加 `CHANGELOG.md` 需显式切换规则）。

### 跨故事模式

- **6.1 → 6.2 依赖：** 发布前 MSVC `cargo check` 与测试策略与 6.1 一致，避免 crates.io 用户与交叉编译矩阵脱节。
- **审查分层：** 6.2 的 Blind / Edge / Acceptance 审查结论已写入故事文件，无阻塞项。
