## 2. 故事层洞察

### Story 4.1（宿主沙箱 rustup/cargo）

- **主路径：** `run_rust_tool_builtin` → `SessionHolder::Host` → `sandbox::run_rust_tool`（导出 → 执行 → 回写）。
- **可诊断性：** 空 PATH 下补充集成测试，断言 `cargo`/`rustup` 缺失的 stderr 与 `BuiltinError` 一致。
- **边界：** 不侵入 γ/β；Linux `DEVSHELL_RUST_MOUNT_NAMESPACE` 等保持 `#[ignore]` 专用环境。
- **工程卫生：** 审查中强调勿将本地 **`.dev_shell.bin`** 漂移混入功能提交。

### Story 4.2（Unix γ / Lima）

- **路径核对：** `limactl_ensure_running`、Mode S 下 push/pull、`ensure_ready` 与 guest 探测与文档一致。
- **测试：** `lima` 缺失时可诊断；`try_session_rc_or_host` 降级到 Host 且 stderr 含 `VM unavailable`。
- **并发：** `vm_env_lock` 串行化 PATH/VM 环境类测试。

### Story 4.3（Windows β / 侧车协议）

- **协议：** handshake `version:1`、单行 JSON 成帧、`op:error` → `VmError::Ipc`；TCP mock 覆盖 handshake / 错误帧 / 非 JSON 前缀。
- **结构：** `BetaSession` 迁至 `session.rs`，模块边界更清晰。
- **验证：** 默认 CI 需带 `--features beta-vm` 跑全量相关测与 clippy。

### Story 4.4（关 VM 与宿主降级）

- **配置真值表：** `DEVSHELL_VM=off`、`backend=host/auto/空` → `SessionHolder::Host`。
- **稳定性：** 引入 **`test_support::vm_env_mutex()`**，`vm` 与 `session_gamma` 测试共享，降低 env 并行干扰。
- **产品说明：** Unix 入口 `try_session_rc_or_host` vs 非 Unix `try_session_rc` 差异在故事内**记录**，未擅自改行为。

### Story 4.5（Mode P 与会话元数据）

- **FR24/FR25：** `workspace_mode_effective`、`is_guest_primary`、`session_store` 路径优先级与 `requirements §1.1` 对齐；guest-primary 退出写 **`.cargo-devshell/session.json`**，不写 legacy bin。
- **测试：** EOF 触发 `save_on_exit` 的集成测；guest-primary REPL 测对 **`beta-vm`** 加 `#[cfg(feature = "beta-vm")]`，避免 `default-features = false` 构建失败。
- **边界提醒：** `.todo.json` 仍在宿主 cwd（设计 §11），本 Epic 未改为 guest 内路径。

### 跨故事模式

- **重复主题：** 全局 env / PATH 测试必须**串行化**或锁；否则非确定性 flake。
- **重复主题：** 本地生成物 **`.dev_shell.bin`** 与功能变更分离，避免脏提交。
- **分层清晰：** 4.1 宿主、4.2 γ、4.3 β、4.4 降级、4.5 Mode P，故事边界在 `epics` 与实现中基本对齐，回归以 `xtask-todo-lib`（及 `beta-vm`）为主。
