## 2. 故事层洞察

### Story 2.1（统一 xtask 开发者子命令）

- **`XtaskSub`** 枚举与 **`docs/requirements.md` §4** 对齐；补 **`ghcr`**、**`lima-todo`** 等遗漏项。
- **新增** `xtask/tests/xtask_help.rs`：顶层与各子命令 `--help` smoke。
- **回归：** `cargo test -p xtask --test integration`、clippy **`-D warnings`**。

### Story 2.2（Git 辅助）

- **行为矩阵：** `add`（多路径暂存）、`pre-commit`（`.githooks/pre-commit`）、`commit`（`-m` 等）。
- **NFR-I1：** 非仓库、hook 缺失、`git` 不在 PATH 等路径有测试；**`git add --help`** 与实现策略一致（「common project paths」等）。
- **实现边界：** 通过宿主 `git`/`sh`，不内嵌 libgit2。

### Story 2.3（发布辅助）

- **`PublishArgs::dry_run`（`--dry-run`）：** 仅 `cargo publish -p xtask-todo-lib --dry-run --registry crates-io`，不改版本、不 git 操作。
- **默认一键发布：** `git status --porcelain` 洁净检查前置；步骤与 **`docs/publishing.md` §2.4** 对齐。
- **文档：** `requirements` §4、`publishing.md` 补充 `--dry-run` 示例。
- **备注：** 全量 `cargo test -p xtask` 中既有不稳定用例（如 `lima_todo` smoke）在故事记录中标注与本次无关。

### Story 2.4（一键验收）

- **`run_all_checks`** 与 **`docs/acceptance.md` §8`** 摘要对齐；**`manual_skip_rows`** 补 **D9**（Windows β + Podman 全链路）及报告覆盖说明。
- **退出码：** 存在 **FAIL** → 非零；**SKIP** 不单独导致失败。
- **边界：** 不将需 Lima/Podman 全链路的项改为本命令内自动执行。

### Story 2.5（提交前检查对齐 CI）

- **对照矩阵：** pre-commit 与 **`.github/workflows/ci.yml`** 逐项标注「仅本地 / 仅 CI / 双方」；**`RUSTDOCFLAGS=-D warnings`** 在 CI **doc** 步与 pre-commit 对齐。
- **故意差异：** MSVC 交叉检查主要在本地 pre-commit/acceptance；**`cargo build`** 在 CI、不在 pre-commit 等，写入 **`requirements` §7.2** / workflow 注释。

### 跨故事模式

- **文档真源：** `publishing.md`、`acceptance.md` §8、**`requirements` §4/§7.2** 与代码**逐步对照**是验收主手段。
- **审查重复项：** 多则故事审查修正 **`sprint-status.yaml` 文件头 `# last_updated` 与 YAML `last_updated` 不一致**——与 Epic 3 回顾 **E3** 同主题。
- **与 Epic 3：** devshell（Epic 3）依赖稳定的 **xtask** 入口与发布/验收叙事；Epic 2 为后续 **cargo devshell** 文档与安装路径提供维护面。
