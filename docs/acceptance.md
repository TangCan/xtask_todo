# 验收文档 (Acceptance)

本文档用于对 [requirements.md](./requirements.md) 中的用户故事与验收标准进行逐项验证，可作为发布或迭代结束时的签字依据。每项验收给出**验证方式**与**预期结果**，通过则在「结果」栏勾选。

---

## 1. 验收说明

- **适用范围**：当前迭代/版本对应的需求范围，以 requirements.md 为准。
- **验证方式**：自动化（测试用例名/命令）或手工步骤；若以测试为主，可注明「由 CI 执行」。
- **结果**：✅ 通过 / ❌ 未通过 / ⏸ 跳过（注明原因）。
- **签字**：验收通过后由产品或项目负责人签字及日期（可写在文末或单独附录）。
- **测试用例**：与各验收项对应的测试用例见 [test-cases.md](./test-cases.md)。

## 2. Todo 领域验收

### 2.1 US-T1：创建待办

| # | 验收标准 | 验证方式 | 预期结果 | 结果 |
|---|----------|----------|----------|------|
| T1-1 | 给定任意有效标题，调用创建接口时，应返回可识别的待办实例（如 id） | 单元/集成测试：对非空标题调用 `TodoList::create`，断言返回 `Ok(id)`，且 `list()` 中包含该项 | 返回 `Ok(TodoId)`，列表中有一条 title 匹配、id 一致的 Todo | |
| T1-2 | 给定空标题或非法输入，调用创建接口时应返回明确错误且不产生有效待办 | 测试：对空字符串（及约定非法输入）调用 `create`，断言返回 `Err`；再调用 `list()` 断言条数未增加 | 返回 `Err(TodoError::InvalidInput)`（或等价），列表长度不变 | |

**建议测试位置**：`crates/todo/tests/` 或 `crates/todo/src/` 内单元测试。

| # | 验收标准 | 验证方式 | 预期结果 | 结果 |
