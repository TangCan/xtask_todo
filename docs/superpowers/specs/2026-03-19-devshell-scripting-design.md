# dev_shell 类 Linux Shell 脚本机制 — 设计说明

**日期**：2026-03-19

**状态**：设计已获用户逐节通过（§1 目标与范围、§2 脚本语法与语义、§3 架构与入口、§4 测试与文档）。

## 1. 目标与范围（§1 已通过）

- **目标**：在 dev_shell 内提供「类 Linux Shell」的脚本能力，用于多步任务自动化；仅作用于 dev_shell 内置命令与同一 VFS，不调用宿主 shell。
- **范围**：
  - 支持：从文件运行脚本、注释与空行、变量（`VAR=value` 与 `$VAR`/`${VAR}` 展开）、条件（`if`）、循环（`for`/`while`）、遇错即停（如 `set -e`）、以及可选的 `source <file>`（在同一 VFS 或宿主路径读文件执行）。
  - 不实现：管道/重定向的语法扩展（沿用现有 pipeline 语法）、后台任务、复杂算术表达式（条件以「命令退出码」或简单字符串比较为主）。
- **与 REPL 的关系**：交互式 REPL 保持「一行一 pipeline」；脚本模式为「解析整个脚本文件 → 解释执行」。可选：REPL 中支持 `source script.dsh` 或 `./script.dsh` 以在交互中跑脚本。

## 2. 脚本语法与语义（§2 已通过）

### 2.1 词法

- 行尾 `\` 表示续行。
- `#` 到行尾为注释；空行忽略。
- 字符串与现有 pipeline 一致，不强制引号；若需要可后续支持 `"..."` 内不展开变量或 `'...'` 字面量。

### 2.2 变量

- **赋值**：`NAME=value`，value 为行内剩余部分 trim，可含空格。
- **展开**：命令参数中 `$VAR`、`${VAR}` 展开；未定义可约定展开为空或报错。
- **作用域**：脚本全局（单层）；后续可考虑局部。

### 2.3 命令

- 与当前一致：一行一条 pipeline（命令 + 管道 + 重定向）。
- 执行前先做变量展开，再交现有 `parse_line` / `execute_pipeline`。

### 2.4 控制流

- **if**：`if command; then` … `fi`（或单行 `if command; then cmd; fi`）。条件为 command 的退出码（0 为真）。可选 `else` / `elif`。
- **for**：`for VAR in a b c; do` … `done`。循环体为多行命令。
- **while**：`while command; do` … `done`。条件为 command 退出码。

### 2.5 source

- `source path` 或 `. path`。
- 在当前 VFS 或宿主路径读取 path 对应文件内容，按脚本解析并执行（同一 VFS、同一变量表）。
- 需防递归（例如 source 深度上限）。

### 2.6 遇错即停

- 脚本级选项（如首行 `set -e` 或命令行参数 `-e`）。
- 开启时，某条命令（非 if/while 条件）退出非零则立即终止脚本并返回该退出码。

### 2.7 首期可实现子集

- 先实现：if / for / while + 变量 + source；再补 set -e 及可选语法细节。

## 3. 架构与入口（§3 已通过）

### 3.1 入口

- `dev_shell script.dsh` 或 `dev_shell -f script.dsh`：脚本文件为参数。
- `dev_shell -e script.dsh`：带 `set -e` 执行脚本。
- 未给脚本文件时：保持现有交互 REPL（或从 stdin 读，与现有一致）。

### 3.2 架构

- **脚本解析**：读入脚本文件 → 词法/语法 → 脚本 AST（或步骤列表）。
- **解释器**：遍历 AST，维护变量表与「遇错即停」标志；遇到「执行命令」节点时先做变量展开，再调用现有 pipeline 解析与 `execute_pipeline`；遇到 source 时递归解析并执行（带深度限制）。
- **与现有代码**：不替换现有 REPL 的 `process_line`，新增「脚本执行」路径；脚本中的「命令」仍通过现有 parser + `execute_pipeline` 执行，保证与交互行为一致。

### 3.3 错误处理

- **解析错误**（语法错误、未闭合的 if/for/while）：报错并退出。
- **运行时错误**（命令失败、source 文件不存在）：按「遇错即停」决定是否终止；终止时进程退出码为非零（例如 1 或最后失败命令的码）。

## 5. 实现时要点摘要

1. **脚本解析模块**：词法（续行、注释、空白）+ 语法（赋值、命令、if/for/while、source）→ AST 或步骤列表；与现有 pipeline 解析协同（命令行先展开变量再交 `parse_line`）。
2. **解释器**：维护变量表、set -e 状态、source 深度；按 AST 执行；命令节点 → 变量展开 → `execute_pipeline`；分支/循环按条件与体执行。
3. **入口**：`run_main_from_args` 识别 `-f`/`-e` 与脚本文件参数，走脚本执行路径；否则保持现有 REPL。
4. **可选**：REPL 中识别 `source path` 或 `./path`，在同一 VFS 下执行该脚本（共享变量表与深度限制）。
5. **不实现**：管道/重定向语法扩展、后台任务、复杂算术；首期可简化 else/elif 或引号规则，在文档中说明。
